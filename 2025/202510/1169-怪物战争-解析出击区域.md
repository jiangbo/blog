# 1169-怪物战争-解析出击区域

## 目标

解析地图中的出击区域，把近战和远程放置点保存下来，供后续出击系统使用。

## 环境

- Time 2026-04-25
- Zig 0.15.1

## 参考

1. <https://www.bilibili.com/video/BV1jf9XYQEhW>

## 想法

实现了出击点的解析，并且使用 debug 在地图上绘制了出来，便于观察。

## map.zig

瓦片数据中已经配置了 `place` 属性。这里先定义一个简单的数据结构保存解析结果。

```zig
pub const PlaceKind = enum { melee, ranged };
pub const Place = struct {
    position: zhu.Vector2,
    size: zhu.Vector2,
    kind: PlaceKind,
};

pub var places: std.ArrayList(Place) = .empty;
```

解析对象层时检查瓦片属性，带有 `place` 属性的对象会作为出击区域保存。

```zig
for (data.layers) |*layer| {
    if (std.mem.eql(u8, "path", layer.name)) {
        parsePathLayer(layer);
    } else {
        switch (layer.type) {
            .tile => parseTileLayer(layer),
            .object => parseObjectLayer(layer),
            else => unreachable,
        }
    }
}

std.log.info("placement count: {}", .{places.items.len});
```

解析对象时，根据对象的 `gid` 找到对应瓦片。如果瓦片带有 `place` 属性，就保存到 `places`，然后继续执行对象绘制逻辑。

```zig
const pos = object.position.addY(-object.size.y);
if (placeKind(tile.?)) |kind| {
    places.append(zhu.assets.allocator, .{
        .position = pos,
        .size = object.size,
        .kind = kind,
    }) catch @panic("oom, can't append place");
}
```

`placeKind` 只关心 `place` 属性。

```zig
fn placeKind(tile: *const tiled.Tile) ?PlaceKind {
    for (tile.properties) |prop| {
        if (!prop.is("place")) continue;
        if (std.mem.eql(u8, prop.value.string, "melee")) return .melee;
        if (std.mem.eql(u8, prop.value.string, "range")) return .ranged;
    } else return null;
}
```

释放地图资源时，也要释放 `places`。

```zig
pub fn deinit() void {
    tileVertexes.deinit(zhu.assets.allocator);
    animations.deinit(zhu.assets.allocator);
    places.deinit(zhu.assets.allocator);
    paths.deinit(zhu.assets.allocator);
}
```

调试时，在地图绘制最后把解析出来的区域画出来，方便确认位置和数量。

```zig
for (places.items) |place| {
    batch.debugDraw(.init(place.position, place.size));
}
```

## 效果

启动游戏后，日志会输出解析到的出击区域数量。画面上会用调试颜色标出解析出来的区域，后续出击系统可以直接使用 `map.places` 判断可放置位置。
