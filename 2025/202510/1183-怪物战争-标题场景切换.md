# 1183-怪物战争-标题场景切换

## 目标

添加标题场景作为游戏启动入口，实现标题场景与战斗场景之间的切换。

## 环境

- Time 2026-05-03
- Zig 0.15.1

## 参考

1. <https://www.bilibili.com/video/BV1GUBHBCEgt/>

## 想法

可以切换场景。目前是main.zig在管理场景的切换，我的想法是scene来作为场景管理器，先就这样。

## context.zig

新增场景状态枚举和延迟切换变量。`SceneState` 使用 `title` 和 `battle` 两个值。

```zig
pub const SceneState = enum { title, battle };
pub var currentScene: SceneState = .title;
pub var pendingScene: ?SceneState = null;
```

地图数组索引 0 是标题地图，关卡从索引 1 开始，`levelIndex` 直接取
`contextZon.level` 的值。

```zig
pub var levelIndex: usize = 0;

pub fn init() void {
    if (contextZon.level == 0) @panic("level must start at 1");
    levelIndex = contextZon.level;
    reset();
}
```

## title.zig

新增标题场景模块。`init`/`deinit` 为空，`enter`/`exit` 负责加载和释放标题地图。

```zig
pub fn enter() void {
    map.init(0);
}

pub fn exit() void {
    map.deinit();
}

pub fn update(delta: f32) void {
    map.update(delta);
}

pub fn draw() void {
    map.draw();
}
```

## map.zig

`maps` 数组添加标题地图（索引 0），改为 `pub` 供外部访问。

```zig
pub const maps = [_]tiled.Map{
    @import("zon/title.zon"),
    @import("zon/level1.zon"),
};
```

## scene.zig

将 `init` 中的地图和生成初始化移到 `enter`，地图释放移到 `exit`。
`init`/`deinit` 只处理模块生命周期，`enter`/`exit` 处理场景切换。

```zig
pub fn init() void {}

pub fn deinit() void {
    spawn.deinit();
}

pub fn enter() void {
    ctx.reset();
    map.init(ctx.levelIndex);
    spawn.init();
}

pub fn exit() void {
    map.deinit();
}
```

## main.zig

`init` 调用所有模块的 `init` 后进入标题场景。
`frame` 开头检查 `pendingScene`，通过 `switchScene` 处理切换。

```zig
pub fn init() void {
    // ...引擎初始化...
    gui.init();
    ctx.init();
    scene.init();
    hud.init();
    title.init();
    title.enter();
}
```

`switchScene` 先退出旧场景，重置注册表，再进入新场景。

```zig
fn switchScene(s: ctx.SceneState) void {
    ctx.pendingScene = null;
    if (ctx.currentScene == .battle) {
        scene.exit();
    } else {
        title.exit();
    }
    registry.reset();

    ctx.currentScene = s;
    if (s == .battle) {
        scene.enter();
        hud.arrangeUnits();
    } else {
        title.enter();
    }
}
```

`frame` 根据 `currentScene` 分支调用对应场景的 update/draw。

```zig
pub fn frame(delta: f32) void {
    if (ctx.pendingScene) |s| {
        switchScene(s);
    }
    if (ctx.currentScene == .battle) {
        const scaled = delta * ctx.timeScale;
        gui.update(&registry, scaled);
        ctx.update(scaled);
        scene.update(&registry, scaled);
        hud.update(delta);
        zhu.batch.beginDraw(tiled.backgroundColor orelse .black);
        scene.draw(&registry);
        hud.draw();
    } else {
        gui.update(&registry, delta);
        title.update(delta);
        zhu.batch.beginDraw(tiled.backgroundColor orelse .black);
        title.draw();
    }
    zhu.batch.flush();
    gui.draw(&registry);
    zhu.batch.commit();
}
```

## gui.zig

`update` 根据场景分支：标题场景显示按钮，战斗场景显示实体信息。
`draw` 同样分支：标题场景显示标题文字，战斗场景显示信息面板。

标题按钮中，"开始游戏"设置 `pendingScene`，"退出游戏"调用
`zhu.window.exit()`。战斗场景设置面板新增"返回标题"按钮。

```zig
fn renderTitleButtons() void {
    // ...
    if (gui.igButton("开始游戏")) {
        ctx.pendingScene = .battle;
    }
    // ...
    if (gui.igButton("退出游戏")) {
        zhu.window.exit();
    }
}
```

使用 `pendingScene` 延迟切换，避免在帧中间改变场景状态导致渲染错误。

## 效果

程序启动进入标题场景，显示标题地图和按钮。点击"开始游戏"进入战斗，
战斗中点击"返回标题"回到标题场景。
