# 1178-怪物战争-技能增益生效

## 目标

让主动技能在施放后真正修改单位属性，并在持续时间结束后恢复。

## 环境

- Time 2026-04-28
- Zig 0.15.1

## 参考

1. <https://www.bilibili.com/video/BV1E8mCBBEmc/>

## 想法

实现技能Buff，感觉这种先计算增益，再减益，有精度损耗啊。

## component.zig

`Stats` 的基础属性改成浮点数。

```zig
pub const Stats = struct {
    health: f32,
    maxHealth: f32,
    attack: f32,
    defense: f32,
    level: f32 = 1,
    rarity: f32 = 1,
};
```

技能增益中的属性倍率是一组字段一起变化，因此把这部分逻辑放在 `Buff` 里。

```zig
pub const Buff = struct { // 技能增益倍率
    health: f32 = 1,
    attack: f32 = 1,
    defense: f32 = 1,
    range: f32 = 1,
    interval: f32 = 1,
    costRecovery: f32 = 0,

    pub fn multiplyStats(self: Buff, stats: *Stats) void {
        stats.health *= self.health;
        stats.maxHealth *= self.health;
        stats.attack *= self.attack;
        stats.defense *= self.defense;
    }

    pub fn divideStats(self: Buff, stats: *Stats) void {
        stats.health /= self.health;
        stats.maxHealth /= self.health;
        stats.attack /= self.attack;
        stats.defense /= self.defense;
    }
};
```

这里只给 `Stats` 提供方法。攻击范围和攻击间隔都只有一个字段，直接在系统里乘除更清楚。

## skill.zig

施放技能时，给实体应用 buff。

```zig
skill.durationTimer = 0;
if (view.tryGetPtr(entity, com.Stats)) |stats| {
    skill.buff.multiplyStats(stats);
}
if (view.tryGetPtr(entity, com.attack.Range)) |range| {
    range.v *= skill.buff.range;
}
if (view.tryGetPtr(entity, com.attack.CoolDown)) |coolDown| {
    coolDown.v *= skill.buff.interval;
}
view.remove(entity, com.skill.Ready);
view.add(entity, com.skill.Active{});
```

持续时间结束后，把属性除回去，然后移除 `Active`，技能重新进入冷却。

```zig
skill.durationTimer = 0;
skill.coolDownTimer = 0;
if (view.tryGetPtr(entity, com.Stats)) |stats| {
    skill.buff.divideStats(stats);
}
if (view.tryGetPtr(entity, com.attack.CoolDown)) |coolDown| {
    coolDown.v /= skill.buff.interval;
}
if (view.tryGetPtr(entity, com.attack.Range)) |range| {
    range.v /= skill.buff.range;
}
view.remove(entity, com.skill.Active);
```

这样技能系统仍然只负责状态推进：

```zig
pub fn update(reg: *zhu.ecs.Registry, delta: f32) void {
    updateCast(reg);
    updateCoolDown(reg, delta);
    updateDuration(reg, delta);
}
```

## health.zig

属性改成浮点数后，伤害计算也同步改成浮点。

```zig
const damage = attack - stats.defense;
stats.health -= @max(damage, attack / 10);
```

血条比例也不再需要整数百分比工具函数，直接计算即可。

```zig
const percent = stats.health / stats.maxHealth;
```

## gui.zig

GUI 上仍然按整数样式显示属性，只是格式从 `%d` 改成 `%.0f`。

```zig
_ = gui.igText("生命值: %.0f / %.0f", stats.health, stats.maxHealth);
_ = gui.igText("攻击力: %.0f", stats.attack);
_ = gui.igText("防御力: %.0f", stats.defense);
```

这样玩家看到的面板还是整数，不会因为内部改成浮点数而显示出很多小数。

## target.zig

治疗目标选择中，最低血量也要从整数哨兵值改成浮点哨兵值。

```zig
var lowestHealth: f32 = std.math.floatMax(f32);
```
