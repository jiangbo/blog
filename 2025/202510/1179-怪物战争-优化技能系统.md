# 1179-怪物战争-优化技能系统

## 目标

重构技能系统的计时、属性备份和特效管理，减少重复代码，统一 ECS 风格。

## 环境

- Time 2026-05-02
- Zig 0.15.1

## 参考

1. <https://www.bilibili.com/video/BV1E8mCBBEmc/>

## 想法

技能系统终于完成了，感觉挺复杂的。

## component.zig

合并后 `buff` 和 `Stats` 类型，默认值全为 1 表示无变化。

```zig
pub const Skill = struct {
    id: SkillEnum,
    name: [:0]const u8,
    description: [:0]const u8,
    passive: bool = false,
    coolDown: f32 = 0,
    duration: f32 = 0,
    displayEntity: ?Entity = null,
    buff: Stats = .{},
    costRecovery: f32 = 0,
};
```

删除 `coolDownTimer` 和 `durationTimer`，改用独立的 `zhu.Timer` 组件。

```zig
pub const Timer = zhu.Timer;
```

新增 `Backup` 组件保存原始属性，buff 结束后直接恢复，避免乘除带来的浮点精度损耗。

```zig
pub const Backup = struct { stats: Stats };
```

## skill.zig

原来的 `updateCoolDown` 和 `updateDuration` 各自维护计时器，合并为一个 `updateTimer`。

```zig
pub fn update(reg: *ecs.Registry, delta: f32) void {
    updateCast(reg);
    updateTimer(reg, delta);
    updateCostRecovery(reg, delta);
    updateDisplay(reg);
}
```

`updateTimer` 只迭代有 `Timer` 组件的实体，没结束就跳过，结束了再根据 `Active` 判断该做什么。

```zig
fn updateTimer(reg: *ecs.Registry, delta: f32) void {
    var view = reg.view(.{com.skill.Timer});
    while (view.next()) |entity| {
        const timer = view.getPtr(entity, com.skill.Timer);
        if (!timer.isFinishedOnceUpdate(delta)) continue;

        const skill = view.getPtr(entity, com.skill.Skill);
        if (view.has(entity, com.skill.Active)) {
            // 持续结束：恢复属性，切回冷却计时
            if (view.tryGetPtr(entity, com.skill.Backup)) |backup| {
                const health = view.get(entity, com.Stats).health;
                view.getPtr(entity, com.Stats).* = backup.stats;
                view.getPtr(entity, com.Stats).health = health;
                view.remove(entity, com.skill.Backup);
            }
            view.remove(entity, com.skill.Active);
            timer.* = .init(skill.coolDown);
        } else {
            // 冷却结束：标记 Ready，移除 Timer
            view.add(entity, com.skill.Ready{});
            view.remove(entity, com.skill.Timer);
        }
    }
}
```

`inline for` 配合 `@typeInfo` 遍历 `Stats` 字段做倍率乘法，新增字段时不需要手动加一行。

```zig
if (view.tryGetPtr(entity, com.Stats)) |stats| {
    view.add(entity, com.skill.Backup{ .stats = stats.* });
    const buff = skill.buff;
    inline for (@typeInfo(com.Stats).@"struct".fields) |field| {
        @field(stats, field.name) *= @field(buff, field.name);
    }
}
view.remove(entity, com.skill.Ready);
view.add(entity, com.skill.Active{});
view.add(entity, com.skill.Timer.init(skill.duration));
```

特效销毁不再直接调用 `destroyEntity`，改为添加 `Dead` 标签，由 death 系统统一处理。

```zig
if (state == null or display.effect != state.?) {
    reg.getPtr(owner, com.skill.Skill).displayEntity = null;
    reg.add(displayEntity, com.Dead{});
    continue;
}
```

## spawn.zig

初始冷却改用 `Timer` 组件，部署时添加半冷却的计时器。

```zig
reg.add(entity, com.skill.Timer.init(skill.coolDown / 2));
```

## gui.zig

冷却进度和剩余时间从 `Timer` 组件读取。

```zig
if (reg.tryGet(entity, com.skill.Timer)) |timer| {
    gui.igProgressBar(timer.progress(), .{ .x = 120, .y = 0 }, null);
}
```

## 效果

技能系统代码量减少，计时逻辑统一用 `zhu.Timer` 组件，属性备份恢复替代乘除避免精度问题。
