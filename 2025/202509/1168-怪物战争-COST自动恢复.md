# 1168-怪物战争-COST自动恢复

## 目标

实现关卡中的 COST 自动恢复，并且用当前 COST 控制头像的灰色遮罩。同时在左上角绘制当前 COST，方便观察它随时间变化。

## 环境

- Time 2026-04-25
- Zig 0.15.1

## 参考

1. <https://www.bilibili.com/video/BV1jf9XYQEhW>

## 想法

感觉现在 AI 越来越强大了，这一次借助 AI 来实现一些东西，看看效果。

## context.zig

COST 是关卡中的全局状态，当前直接放在 `context.zig` 中。

```zig
const INITIAL_COST: f32 = 10;
const COST_GEN_PER_SECOND: f32 = 1;
const INITIAL_HOME_HEALTH: i32 = 5;

fn playerCost(class: PlayerEnum) u8 {
    return spawn.playerZon[@intFromEnum(class)].cost;
}

pub var cost: f32 = INITIAL_COST;
pub var homeHealth: i32 = INITIAL_HOME_HEALTH;
pub var enemyCount: u32 = 0;
pub var enemyArrivedCount: u32 = 0;
pub var enemyKilledCount: u32 = 0;
```

每帧恢复 COST：

```zig
pub fn update(delta: f32) void {
    cost += COST_GEN_PER_SECOND * delta;
}
```

判断和扣除 COST：

```zig
pub fn canAffordCost(value: u8) bool {
    return cost >= @as(f32, @floatFromInt(value));
}

pub fn spendCost(value: u8) void {
    if (!canAffordCost(value)) return;
    cost -= @floatFromInt(value);
}
```

同时保留了按角色类型判断和扣除的函数，后续出击时可以继续使用。

```zig
pub fn canAfford(class: PlayerEnum) bool {
    return canAffordCost(playerCost(class));
}

pub fn spend(class: PlayerEnum) void {
    spendCost(playerCost(class));
}
```

## hud.zig

头像遮罩从固定判断改成根据当前 COST 判断，COST 不足时头像会盖灰，恢复到足够以后遮罩会自动消失。

```zig
if (!ctx.canAffordCost(unit.cost)) {
    batch.drawRect(.init(unit.position, uiZon.frameSize), .{
        .color = .rgba(0, 0, 0, 0.2),
    });
}
```

为了方便观察当前 COST，在左上角绘制一个简单数字，只显示整数部分。

```zig
const currentCost: u32 = @intFromFloat(@floor(ctx.cost));
text.drawColor("COST:", .xy(24, 24), .yellow);
text.drawNumberColor(currentCost, .xy(120, 24), .white);
```

## 效果

游戏启动后，左上角会显示当前 COST。COST 会从 10 开始，每秒增加 1 点。
底部头像如果消耗大于当前 COST，会显示灰色遮罩；当 COST 增长到足够时，遮罩会自动消失。
